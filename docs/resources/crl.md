# azion_crl (Resource)

Creates a Certificate Revocation List (CRL) resource. CRLs are used to verify the revocation status of X.509 digital certificates.

## Example Usage

```terraform
resource "azion_crl" "example" {
  crl = {
    name   = "My Certificate Revocation List"
    issuer = "CN=My Certificate Authority,O=My Organization,C=US"
    crl    = "-----BEGIN X509 CRL-----\nMIIBpzCCAVMCAQEwDQYJKoZIhvcNAQELBQAwGjEYMBYGA1UEAwwPVGhpcyBpcyBh\nIHRlc3QgQ0EXDTI0MDEwMTAwMDAwMFoXDTI1MDEwMTAwMDAwMFqgDjAMMAoGA1Ud\nFAQDAgEBMAoGA1UdFAQDAgEAMA0GCSqGSIb3DQEBCwUAA4IBAQC0test\n-----END X509 CRL-----"
  }
}
```

### Nested Schema for `crl`

Required:

- `crl` (String) The certificate revocation list content in PEM format.
- `issuer` (String) The issuer of the certificate revocation list (the distinguished name of the CA that issued this CRL).
- `name` (String) Name of the certificate revocation list.

Optional:

- `active` (Boolean) Indicates if the certificate revocation list is active. Note: This field cannot be set to false.

Read-Only:

- `created_at` (String) Timestamp of the certificate revocation list creation on the platform.
- `id` (Number) The identifier of the certificate revocation list.
- `last_editor` (String) The last editor of the certificate revocation list.
- `last_modified` (String) Timestamp of the last modification made to the certificate content on the platform.
- `last_update` (String) Timestamp of the last update issued by the certification revocation list issuer.
- `next_update` (String) Timestamp of the next scheduled update from the certification revocation list issuer.
- `product_version` (String) Product version of the certificate revocation list.

## Import

Import is supported using the following syntax:

```shell
terraform import azion_crl.example crl_id
```

The `crl_id` is the numeric identifier of the Certificate Revocation List.
